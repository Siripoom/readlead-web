เอกสารนี้สรุป journey ปัจจุบันของ ReadLead จาก flow ที่มีอยู่ใน prototype: การค้นพบคอนเทนต์, การอ่าน, การสมัครสมาชิก, wallet, การปลดล็อกตอน, และ creator studio

Current-state journey map
Audience
Product, Design, Engineering

Persona Overview

สรุปเป้าหมาย, จุดเริ่มต้น, และผลลัพธ์ที่ถือว่าสำเร็จของแต่ละ role ตาม behavior ปัจจุบันในแอป

Guest

ผู้อ่านทั่วไป
Goal สำรวจเรื่องที่น่าสนใจและทดลองอ่านตอนฟรีก่อนสมัคร
Entry point /, /discover, category links
Success outcome เจอเรื่องที่อยากอ่านต่อ แล้วไป /login หรือ /register

User

สมาชิก
Goal อ่านต่อเนื่อง, ปลดล็อกตอน, เก็บเรื่องโปรด, สนับสนุนนักเขียน
Entry point /login, /register, /dashboard
Success outcome อ่าน/ซื้อ/โหวต/โดเนทได้ และกลับมาดูประวัติใน dashboard

Creator

นักเขียน
Goal สร้างผลงาน, จัดการตอน, ตั้งราคา/โปรโมชัน, ดูรายได้
Entry point /creator, /creator?tab=works
Success outcome ส่งงานเผยแพร่/รอตรวจ, ติดตาม revenue, ขอถอนเงินได้

Role Transition Summary

สรุปการเปลี่ยน role และ route guard ตาม prototype ปัจจุบัน

Guest -> User

ผู้ใช้เริ่มจากการ browse ที่ / หรือ /discover แล้วไป /login หรือ /register. เมื่อ submit form mock auth จะตั้ง role เป็น user ผ่าน RoleContext/localStorage และ redirect กลับหน้าแรก

User/Creator/Admin Access

/dashboard เปิดให้ user, creator, admin. /creator เปิดให้ creator และ admin เท่านั้น. หาก role ไม่ตรง RouteGuard จะแสดง Access Denied พร้อมลิงก์กลับหน้าแรก
